# 前端说明

## 前端主要的功能

主要是三个方面，1系统管理，2组态编辑（仿真和优化都会用到），3大屏展示

### 系统管理

系统管理就是添加用户、设备类型、设备实例，设置点位等等，以及设置告警、规则等等。

渐进式开发，早期只需要设置设备类型、设备实例、实测数据点位。当然鉴权是需要的，早期都不考虑租户之类的。角色也不用考虑那么多。

### 大屏展示

这个就是各种渲染，统计曲线。目前不需要，往后放。

### 组态编辑

这个重要。本质上就相当于是一个系统编辑界面了。

对于仿真，就是开启一个项目，允许新建一个设备的模型（具体就是图标、变量、参数、方程），然后注册这个设备类型。然后可以拖拉拽用设备实例搭建系统，每个设备还允许设置它的参数。

对于优化，还需要设置哪些变量是决策变量等等。

## 技术栈选择

### 管理界面

我们选择使用vue，比较过react和angular等等。vue比较容易理解，也容易看懂维护。所以选择vue。

系统管理方面，可以选择element-plus。看过https://github.com/go-admin-team/go-admin, https://github.com/flipped-aurora/gin-vue-admin等等。可以给我们建议。
